# reports/

Quarto report documenting the data analysis, modeling approach, and
results for the capstone project. Renders to HTML.

```
reports/
├── _quarto.yml   # Quarto project config (jupyter engine: harvest-analysis kernel)
└── report.qmd    # Main report source — executable, pulls from analysis/src
```

`report.qmd` is not static text — its tables and charts are generated by executable
Python code chunks that import `analysis/src/{data,preprocessing,modeling}.py` directly,
the same modules the notebooks use. Rendering it re-runs the real pipeline (split,
leaderboards, correlation filter, test evaluation) rather than pasting in old numbers.

## Rendering

Requires the Quarto CLI (see the project root's instructions for manual `.pkg`
installation) and the `harvest-analysis` Jupyter kernel registered from
`analysis/.venv`:

```bash
cd analysis
source .venv/bin/activate
python -m ipykernel install --user --name harvest-analysis --display-name "Python (harvest-analysis)"
```

Then render:

```bash
cd reports
quarto render
```

Output goes to `reports/_output/`.
